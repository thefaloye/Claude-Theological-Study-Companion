# Contributing to the Theological Study Companion

Thank you for considering a contribution. This project exists to serve pastors, seminarians, and serious students of Scripture with a **theologically consistent, confessionally honest** tool. Because of that, contributions are welcomed within defined boundaries — please read this guide before opening an issue or pull request.

---

## Core Principle

This skill is **not theologically neutral**, and that is by design. It operates from:

- A **grammatical-historical** hermeneutic integrated with **redemptive-historical / biblical theology**
- The **1689 London Baptist Confession of Faith** as primary confessional standard, with the **Westminster Confession** as secondary
- A canon of **historically-tested** lexicons, dictionaries, concordances, and theologians (patristic through modern Reformed)

Contributions that would shift the project outside this framework — whether toward hyper-charismatic theology, hyper-cessationism, liberal-critical methodology, Open Theism, New Age frameworks, or paraphrase-based translations — will not be accepted, regardless of how well-argued or well-sourced. This is a feature of the project, not an oversight. If you are looking for a more theologically eclectic or neutral tool, this project will not be a good fit, and that's fine — fork it.

---

## What We Welcome

### 1. New Reference Works (Lexicons, Dictionaries, Concordances)

If you believe a historically-tested resource is missing from `references/reference-works.md`, open an issue first describing:

- The resource and its publication history
- Its theological orientation (must be evangelical/Reformed-compatible — see exclusions below)
- Where it fits in the existing tiers (primary lexicon, supplement, theological wordbook, etc.)
- Why it adds value beyond what's already covered

**Excluded categories** (will not be accepted): any resource with a Neo-Pentecostal, Charismatic, Hyper-charismatic, or New Age editorial agenda (e.g., Dake's Annotated Reference Bible, Spirit-Filled Life Bible notes).

### 2. New Theologians or Historical Voices

If proposing an addition to `references/theologians-canon.md`, please include:

- Name, dates, and tradition
- Primary works
- A clear statement of **specific relevance** — what theological loci or tasks this voice serves
- Any necessary cautions (as we do for Origen, Tertullian, Aquinas, etc.)

Entries should follow the existing format: tradition/status line, primary works, and a relevance note. Vague or "completeness for its own sake" additions will likely be declined — the project deliberately prioritizes **accuracy of relevance over breadth of citation**.

### 3. Corrections

Factual errors — incorrect dates, misattributed works, mischaracterized positions, broken cross-references — are always welcome. These can go straight to a pull request with a brief explanation.

### 4. Refinements to the Controversy-Handling Protocol

If you believe a doctrine should be added to, removed from, or reframed within the "Theological Controversy Protocol" or "Standing Exclusions" sections of `SKILL.md`, open an issue for discussion first. These sections define the project's confessional boundaries and changes here affect the entire skill's behavior.

### 5. Output Format & Usability Improvements

Suggestions to improve the Structured Study Note format, task-specific guidance, or original language protocol are welcome — especially from pastors and seminarians using the skill in real ministry contexts. Practical feedback from actual use is highly valued.

---

## What We Will Not Accept

- Additions that introduce hyper-charismatic, New Apostolic Reformation, Open Theism, Process Theology, New Age/mystical, or liberal-critical (e.g., JEDP-as-assured-result, Bultmannian) frameworks as legitimate or neutral options
- Support for The Passion Translation (TPT), the Mirror Bible, The Message, The Living Bible, or other paraphrases as primary or comparison translations
- Removal of the 1689 LBCF as the primary confessional standard, or framing it as merely "one option among many" within this skill
- Theologians or sources cited without clear, accurate relevance (padding the canon)

If you're unsure whether a contribution fits, **open an issue first** rather than going straight to a pull request — it saves everyone time.

## How to Contribute

### Reporting Issues

Use the issue templates provided:
- **Propose a Reference/Theologian** — for additions to the canon
- **Bug Report** — for cases where the skill behaves inconsistently with `SKILL.md` (e.g., cites excluded sources, mishandles a contested doctrine, ignores the translation policy)

### Pull Requests

1. Fork the repository and create a branch from `main`
2. Make your changes, following the existing formatting conventions in `SKILL.md` and the `references/` files
3. If you're adding a theologian or reference work, follow the entry format already established for that section
4. Open a pull request with a clear description of **what** changed and **why**, referencing any related issue
5. Be prepared for theological review — changes to confessional boundaries or the canon may take longer and involve discussion

### Style Conventions

- `SKILL.md` should remain under ~500 lines where possible; move detailed content into `references/`
- Use the existing heading hierarchy and table formats
- Theologian entries: **Name (dates)** → Tradition/Status → Primary Works → Relevance
- Reference work entries: **Name (author, date)** → Status/Tier → Description → Usage notes
- Keep tone consistent: precise, reverent, pastoral — avoid casual language in skill instructions

---

## Code of Conduct

Disagreement on theological matters is expected and welcome — within the bounds of charity and the project's stated confessional framework. Personal attacks, mockery of confessional positions, or attempts to relitigate the project's foundational scope in every thread will not be tolerated. Keep discussion focused, sourced, and respectful.

---

## Questions

If you're unsure where something fits, open an issue tagged `question` and we'll help you figure it out.
